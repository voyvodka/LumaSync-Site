## What ships today (v1.2.0)

LumaSync's mode model has three concrete values:

| Mode | Behaviour |
|------|-----------|
| **Off** | No output. USB strip and Hue stream both idle. |
| **Ambilight** | Real-time screen capture drives both targets. See [Screen capture](/docs/ambilight/screen-capture). |
| **Solid** | Single RGB + brightness pushed to USB and Hue. Room fill without screen dependence. |

Mode selection is compact-mode first: the three tiles at the top of the
compact window switch between them with one click. Full-mode users select
the same way from the header mode switcher.

### Off

Truly off — the pipeline stops sampling the screen, serial frames stop
(the strip holds its last output but LumaSync sends no updates), and Hue
stream shuts down. Closing the app window while in Off still leaves the
tray icon running; quit from tray to kill entirely.

### Ambilight

The default interesting mode. All four [ambilight tuning](/docs/ambilight/tuning)
knobs (saturation, smoothing, brightness cap, update rate) apply here.

### Solid

Pick an RGB colour and brightness; both targets push the same value. Good
for:

- Constant bias lighting while reading / coding
- A specific mood colour for a room
- Debugging strip wiring (send pure red to check every LED)

Colour changes debounce at ~50 ms so a rapidly-moving colour picker doesn't
flood the serial link or the Hue stream.

## Presets

**[TBD: verify]** A preset system — pinning a mode + its parameters with a
name — is called out in the site navigation but is not a shipped feature in
v1.2.0. Today, changing saturation / smoothing / update rate persists
globally via the `shellStore` facade to `~/.config/lumasync/app.json`, but
there's no UI to save multiple named configurations and switch between
them.

When the preset system ships, this page will document:

- How to save the current settings as a named preset
- How to switch between presets (keyboard shortcut + UI)
- Where presets live (app.json key under `presets[]`)
- Import / export for sharing

Until then, if you want a "reading mode" vs "movie mode" split, the
pragmatic workflow is to flip Ambilight ↔ Solid and adjust the solid
colour / brightness for reading.

## What the contract allows

The lighting-mode contract
(`src/features/mode/model/contracts.ts`, `LIGHTING_MODE_KIND`) is a
closed enum:

```ts
LIGHTING_MODE_KIND = {
  OFF: "off",
  AMBILIGHT: "ambilight",
  SOLID: "solid",
} as const
```

Adding new modes (rainbow, audio-reactive, etc.) requires a contract
change — shipping scene presets without a new mode kind is possible by
stacking parameters on top of existing Ambilight / Solid modes.
