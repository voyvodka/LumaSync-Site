## 2026-04-27 - [JSON-LD Serialization XSS]

**Vulnerability:** The JSON-LD schema objects generated in `src/lib/seo.ts` and inserted via `JSON.stringify(schema)` inside a `<script type="application/ld+json">` tag using Astro's `set:html` attribute were vulnerable to Cross-Site Scripting (XSS). If the title or description (e.g. from an Astro Markdown page's frontmatter) contained a string like `</script><script>alert(1)</script>`, `JSON.stringify` would correctly encode the characters for JSON, but the browser would see the `</script>` tag and break out of the script tag context, executing the payload. **Learning:** `JSON.stringify` does not escape HTML control characters `<` and `>`. When injecting JSON directly into a script tag using an unescaped raw HTML mechanism (like Astro's `set:html`), you must manually replace these characters to avoid breaking out of the `<script>` context. **Prevention:** Always escape `<` to `\u003c`, `>` to `\u003e`, and `&` to `\u0026` after using `JSON.stringify` when placing its output directly inside a `<script>` tag.
